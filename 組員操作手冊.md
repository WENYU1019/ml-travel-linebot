# 組員操作手冊

本文件提供組員開發、測試、上傳與協作規範。專案主體是根目錄的 `app.py`，請不要再建立第二層專案資料夾。

## 開發環境

- 作業系統：Windows
- 終端機：PowerShell
- Python：建議 Python 3.11
- 虛擬環境：只使用根目錄的 `venv/`
- 套件管理：`requirements.txt`
- 主要套件：
  - `flask`
  - `line-bot-sdk`
  - `python-dotenv`
  - `groq`

## 第一次設定

```powershell

```

再打開 `.env` 填入自己的 API key 與 LINE token。

## API key 取得方式

### LINE

1. 建立或登入 LINE Official Account。
2. 啟用 Messaging API。
3. 進入 LINE Developers Console。
4. 取得 `Channel secret`。
5. 在 Messaging API 頁面發行 channel access token。
6. 將 webhook URL 設定為服務網址加 `/callback`。

需填入 `.env`：

```env
LINE_CHANNEL_ACCESS_TOKEN=...
LINE_CHANNEL_SECRET=...
```

### Groq

1. 登入 GroqCloud Console。
2. 進入 API Keys 頁面。
3. 建立 API key。
4. 填入 `.env`。

需填入 `.env`：

```env
GROQ_API_KEY=...
GROQ_MODEL=llama-3.3-70b-versatile
```

## 上傳規範

- 一次 commit 只處理同一類事情，例如文件、功能、修 bug 不要混在一起。
- commit message 建議使用：
  - `docs: 說明文件更新`
  - `feat: 新增功能`
  - `fix: 修正錯誤`
  - `refactor: 程式整理`
  - `test: 測試更新`
- 不要上傳 `.env`。
- 不要上傳新的子資料夾 venv。
- 不要把程式主體放回 `ai_linebot_core/` 這種子專案資料夾。
- 文件放 `docx/`，前端放 `fronted/`，程式功能放第一層功能資料夾。
- 修改別人的檔案前，先確認目前分工，避免互相覆蓋。

## 常見問題

### 啟動時找不到套件

先確認已啟用根目錄的 `venv`，再重新安裝：

```powershell
venv\Scripts\activate
python -m pip install -r requirements.txt
```

### LINE webhook 收不到訊息

請檢查：

- LINE Developers Console 的 webhook URL 是否為公開 HTTPS URL。
- URL 是否以 `/callback` 結尾。
- `.env` 的 `LINE_CHANNEL_ACCESS_TOKEN` 和 `LINE_CHANNEL_SECRET` 是否正確。
- Flask 服務是否正在執行。

### Groq 無法使用

請檢查：

- `.env` 是否有 `GROQ_API_KEY`。
- API key 是否有效。
- `GROQ_MODEL` 是否為 Groq 可用模型。
- 若沒有 Groq key，系統會改走 fallback，但結果可能較簡略。
